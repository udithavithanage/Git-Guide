Git Workflow Guide

Step-by-step guide for updating your branch with the latest changes from the main branch before pushing your own changes.

Steps to Update Your Branch

1. Commit Your Local Changes

First, commit any changes in your working directory to avoid conflicts. 

git add .

git commit -m "Your commit message"

2. Switch to Your Branch

Ensure youâ€™re on your current working branch (the one you want to push). 

git checkout your-branch-name

3. Pull Latest Changes from the Main Branch

You need to fetch and merge the latest updates from the main branch into your current branch. 

git pull origin main

If there are no conflicts, it will merge automatically. If there are conflicts, Git will prompt you to resolve them.

4. Resolve Any Conflicts (If Any)

If conflicts occur during the merge, Git will mark the files as conflicted. Open those files, resolve the conflicts manually, then stage and commit the resolved files: 

git add conflicted-file

git commit -m "Resolved merge conflicts"

5. Push Your Changes to the Remote Repository

After successfully updating your branch and resolving any conflicts, push your changes to the remote repository. 

git push origin your-branch-name

Summary:

  • Commit local changes.
  • Pull the latest changes from the main branch (git pull origin main).
  • Resolve conflicts (if any).
  • Push your updates (git push origin your-branch-name).
